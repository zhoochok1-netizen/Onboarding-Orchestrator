# История промптов — Claude Code (Opus / Sonnet)

Документ фиксирует промпты из сессии разработки с Claude Code. Промпты сгруппированы по этапам, очищены от дублей и отформатированы.

---

## Этап 1. Инициализация проекта

**Промпт:**

> Прочитай тех задание в формате HTML. Начни реализовывать проект с моковыми данными. Архитектура: основная папка проекта — `testing`, внутри — `frontend` и `backend`. Стек: React на фронте, Python + FastAPI на бэке. Стили базовые, но красивые.

**Что сделано:**

- Структура проекта: `frontend/` (React 19, CRA) + `backend/` (FastAPI, Uvicorn)
- Pydantic-модели: Employee, OnboardingTemplate, Stage, Task, SLA, ChatMessage
- Mock-данные: 8 сотрудников, 2 шаблона, 3 онбординга, 25 задач, 4 документа базы знаний
- 6 страниц: Dashboard, TaskBoard, Templates, Onboardings, OnboardingDetail, Chatbot, KnowledgeBase
- Sidebar с навигацией, API-клиент на фронте

## Этап 3. Дизайн и UI

> Объединяет несколько итераций по визуалу.

**Промпты:**

> Переделай фронт в более современный стиль. Добавь эффект жидкого стекла (glassmorphism). Нужен TopBar для навигации и профиля. Сайдбар оставить, но переделать в стеклянном стиле.

> Сделай красивую тёмную тему. Минималистичный дизайн. Модалку профиля.

> Все выпадающие списки должны быть стилизованными, не нативными. У нас glassmorphism-концепция, нативные белые дропдауны выглядят чужеродно.

**Что сделано:**

- Glassmorphism: `backdrop-filter: blur()`, полупрозрачные фоны, стеклянные карточки
- TopBar: заголовок страницы, поиск, переключатель темы, профиль с выпадающим меню
- Тёмная тема (`#0a0a0f`) + светлая, переключатель, сохранение в localStorage
- Анимированный градиентный фон с размытыми орбами
- Компонент `CustomSelect` — полностью кастомный dropdown через React Portal, не обрезается `overflow`
- Кастомный скроллбар, selection-стили, плавные переходы

## Этап 4. Ролевая модель

> Объединяет работу над авторизацией и фильтрацией по ролям.

**Промпты:**

> Нужна возможность входа под разными профилями — HR видит все задачи, разработчик только свои. Сделать выбор профиля при входе, без логина/пароля.

> Сайдбар должен меняться под роль. Новичку не нужна статистика по всем онбордингам. Данные должны обновляться мгновенно при смене профиля.

**Что сделано:**

- `ProfileSelector` — экран выбора профиля, сотрудники сгруппированы по ролям
- 5 ролей: HR, Manager, IT, Mentor, Newcomer
- Бэкенд-фильтрация через `X-User-Id` (HR — всё, Manager — свой отдел, остальные — свои задачи)
- Ролевая навигация в Sidebar (разное меню для каждой роли)
- `UserContext` + `ThemeContext` (React Context API)
- Динамическая статистика в футере Sidebar

## Этап 5. AI-чатбот и база знаний

> Объединяет подключение AI, улучшение ответов, кликабельные ссылки и расширение базы знаний.

**Промпты:**

> Увеличь базу знаний и оптимизируй работу с AI-чатом. Когда спрашиваю «что мне нужно сделать» — он должен чётко ответить по моим задачам. Проработай пайплайн с базой знаний и чатом. Markdown в ответах — жирный текст, списки, без звёздочек.

> Если в ответе упоминается задача — прикрепляй ссылку. По клику — модалка с подробностями (статус, SLA, дедлайн, участники, прогресс онбординга).

> Ссылки на документы базы знаний тоже должны быть кликабельными и открывать модалку с содержимым документа, а не вести на общую страницу `/knowledge`.

> Сохраняй переписку с чатботом в рамках профиля пользователя. Ответы по задачам должны быть ёмкими, без воды.

**Что сделано:**

- База знаний: 8 → 15 документов (коммуникации, доступы, оценка, ДМС, Jira, удалёнка и др.)
- OpenAI-совместимый клиент через OpenRouter (Gemini 2.5 Flash)
- Системные промпты с инструкциями по формату ссылок: `task:ID`, `onboarding:ID`, `knowledge:DOC_ID`
- `_build_user_context()` — детальный контекст с TASK_ID и ONB_ID для каждого элемента
- `react-markdown` для рендеринга ответов
- 3 типа кликабельных ссылок → модалки: TaskDetailModal, OnboardingDetailModal, KnowledgeModal
- Препроцессинг markdown: `task:` → `#task--` (обход санитизации react-markdown)
- Сохранение чата в localStorage по `user.id`
- Ролевые подсказки (suggestions) для каждой роли
- Кнопка «Очистить чат»
- Mock-ответы с контекстом пользователя (задачи, просрочки, прогресс)

## Этап 7. Доска задач — UX для HR

> Объединяет доработки доски задач: модалки, статусы, пинг, фильтры.

**Промпты:**

> SLA-точки всегда зелёные — неправильно. Выпадающий список статуса без стилей. HR нужна кнопка «Запустить онбординг».

> HR должен видеть подробности задачи в модалке — кого и как пингануть. Добавь автопинг: за N дней до дедлайна уведомлять ответственных.

> Пинг должен быть у каждого участника задачи (и новичок, и ответственный), а не только у одного.

> Добавь фильтр по SLA на доске задач (в срок / скоро дедлайн / просрочено).

**Что сделано:**

- SLA-точки: выполненные задачи показывают ✓, остальные — цветную точку по `sla_status`
- `CustomSelect` для статуса задачи — цветная пилюлька (фиолет/жёлтый/зелёный/красный)
- Клик по строке задачи → модалка: статус, SLA, дедлайн с обратным отсчётом, участники
- Кнопка «📨 Пинг» у каждого участника (новичок + ответственный), только для HR
- Модалка «⚙️ Автопинг»: тоггл вкл/выкл, выбор дней (1–7), каналы (Slack/Email/В системе), превью
- Фильтр по SLA: 🟢 В срок / 🟡 Скоро дедлайн / 🔴 Просрочено
- Настройки автопинга сохраняются в localStorage

## Этап 9. Шаблоны и создание онбордингов

> Объединяет CRUD шаблонов и редактор задач при создании онбординга.

**Промпты:**

> HR должен иметь возможность создавать новые шаблоны с этапами (день 1, неделя 1, месяц 1) и задачами.

> При создании онбординга HR выбирает шаблон и может добавить или убрать задачи из пайплайна.

> Модалка создания шаблона должна быть шире. Слева блок этапов, справа задачи. Дропдауны не должны обрезаться.

**Что сделано:**

- `POST/PUT/DELETE /api/templates` на бэкенде
- Модалка создания шаблона: двухколоночный layout (этапы слева, задачи справа)
- Редактор этапов: добавление, удаление, редактирование названия и описания
- Добавление задач: название, этап (CustomSelect), роль, дедлайн
- Редактирование и удаление существующих шаблонов (только HR)
- Создание онбординга: выбор шаблона → загрузка задач → чекбоксы вкл/выкл → добавление кастомных задач
- CustomSelect через `ReactDOM.createPortal` — dropdown рендерится в `<body>`, не обрезается `overflow`

## Системные промпты AI

### Чатбот (`CHAT_SYSTEM_PROMPT`)

- Краткие ёмкие ответы (10–15 строк), Markdown
- Обязательные кликабельные ссылки: `[Title](task:TASK_ID)`, `[Name](onboarding:ONB_ID)`, `[Doc](knowledge:DOC_ID)`
- Источники из базы знаний с DOC_ID
- Адаптация ответа под роль пользователя
- Честный fallback: «не знаю — обратитесь к HR»

### Навигатор по базе знаний (`KNOWLEDGE_SYSTEM_PROMPT`)

- Ответ строго на основе документов
- Обязательное указание источника: `[Название](knowledge:DOC_ID)`
- Markdown форматирование
