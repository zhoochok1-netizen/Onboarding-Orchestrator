# История промптов — Onboarding Orchestrator

Документ фиксирует ключевые промпты, которые использовались при разработке проекта с помощью AI-агента (Claude Code).

---

## 1. Инициализация проекта

**Промпт:**
> Прочитай тех задание в формате html. И начни реализовывать проект, пока что с моковыми данными. По архитектуре должно быть так: основная папка проекта (testing) внутри папки frontend и backend. По стеку пишем на реакте фронт, а бэк через питон и фаст апи. Стили так же пока что применяем базовые, но красивые.

**Результат:** Создана структура проекта, backend с FastAPI (models, mock_data, main), frontend с React (6 страниц + Sidebar). Моковые данные: 8 сотрудников, 2 шаблона, 3 онбординга, 25 задач.

---

## 2. Расширение данных

**Промпт:**
> Напихай в наш проект больше данных моковых, и больше юзеров

**Результат:** 8→22 сотрудника, 2→4 шаблона (добавлены продажи и дизайн), 3→10 онбордингов, 25→100 задач, 4→8 документов базы знаний. Добавлены контекстные mock-ответы чатбота (15 тем).

---

## 3. Редизайн UI — Glassmorphism + TopBar

**Промпт:**
> Сделаем фронт более современным. Может быть жидкое стекло немного. Нужен красивый фронт, и наверное нужно добавить топ бар для навигации и профиля. Сайдбар тоже нужно оставить, но чуть его переделать.

**Результат:** Glassmorphism (backdrop-filter blur, полупрозрачные фоны), TopBar с поиском/уведомлениями/профилем, стеклянный Sidebar, анимированный градиентный фон, glass-карточки.

---

## 4. Тёмная тема + минимализм

**Промпт:**
> Сделай темную тему красивую. Сделай модалку красивую о профиле. Дизайн наверное хочется более минималистичный.

**Результат:** Полная тёмная тема (#0a0a0f фон), минималистичные карточки, сдержанные цвета, профильная модалка.

---

## 5. Ролевая модель

**Промпт (собственная доработка):**
> Добавить выбор профиля при входе, ролевую навигацию в Sidebar, фильтрацию данных на бэкенде по X-User-Id.

**Результат:** 5 ролей (HR, Manager, IT, Mentor, Newcomer), ProfileSelector, ролевая навигация (разное меню), бэкенд-фильтрация через _filter_tasks_for_user / _filter_onboardings_for_user. Тёмная/светлая тема.

---

## 6. AI-интеграция

**Промпт (собственная доработка):**
> Подключить AI через OpenRouter, сделать config.py с настройками, ai_client.py с системными промптами, контекстный чат.

**Результат:** OpenAI-совместимый клиент, системные промпты для чата и базы знаний, _build_user_context() для контекста пользователя, fallback на mock-ответы.

---

## 7. Улучшение чата и базы знаний

**Промпт:**
> Увеличим базу знаний и оптимизируем работу с AI чатом. Чтоб он чётко говорил, например я спрашиваю что мне нужно сделать по моим задачам, он мне всё сразу чётко ответил. В целом проработаем весь этот пайплайн с базой знаний и чатом. Более развёрнуто ответить, что и где нужно взглянуть и посмотреть, а так же прикреплять ссылки на базу знаний. И основное, маркдауны в сообщениях от модели, что бы не было этих звёздочек, был нормальный жирный текст.

**Результат:**
- База знаний: 8→15 документов (добавлены коммуникации, доступы, оценка, ДМС, встречи, Jira, удалёнка)
- react-markdown для рендеринга ответов
- Детальный _build_user_context с задачами, просрочками, онбордингами
- Ёмкие mock-ответы с контекстом пользователя

---

## 8. Кликабельные ссылки и модалки

**Промпт:**
> Если мы видим ответ, что какому-то сотруднику нужно что-то сделать, нам наверное нужно прикреплять ссылку на задачку для этого сотрудника, или хотя бы открывать модалку с этой задачкой. Нажимаю на эту ссылку — мы открываем модалку, где видим более подробную информацию, какой процесс и тд.

**Результат:**
- Специальный формат ссылок: `[Задача](task:id)`, `[Имя](onboarding:id)`, `[Документ](knowledge:doc-id)`
- 3 модалки: TaskDetailModal, OnboardingDetailModal, KnowledgeModal
- Препроцессинг markdown → безопасные #-якоря → event delegation для кликов
- Сохранение чата в localStorage по пользователю
- Ролевые подсказки (suggestions) для чата

---

## 9. RAG Pipeline

**Промпт:**
> Давай ещё раз проверим соответствие по ТЗ. Мне дали рекомендации: глобальное виденье проекта, архитектура, README, RAG. Нужно очень хорошо постараться.

**Результат:**
- Собственный RAG pipeline (`rag.py`): chunking → TF-IDF scoring → top-K retrieval
- 15 документов → 86 чанков, 792 токена в словаре
- RAG-контекст в чате вместо полной базы знаний (экономия токенов)
- Docker + docker-compose для деплоя
- Подробный README с архитектурными диаграммами
- Эндпоинт `/api/rag/stats` для мониторинга пайплайна

## Системные промпты AI

### Чатбот (ai_client.py → CHAT_SYSTEM_PROMPT)

Ключевые инструкции модели:
- Ёмкие ответы (10-15 строк для задач)
- Markdown форматирование
- Обязательные ссылки: `[Title](task:TASK_ID)`, `[Name](onboarding:ONB_ID)`, `[Doc](knowledge:DOC_ID)`
- Источники из базы знаний с DOC_ID
- Адаптация под роль пользователя

### Навигатор по базе знаний (KNOWLEDGE_SYSTEM_PROMPT)

- Ответ только на основе документов
- Обязательное указание источника
- Markdown форматирование
