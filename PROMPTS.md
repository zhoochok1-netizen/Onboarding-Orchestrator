# История промптов — Onboarding Orchestrator

Документ фиксирует ключевые промпты, которые использовались при разработке проекта с помощью AI-агента (Cursor, Claude).

---

## Этап 1. Инициализация и архитектура

**Промпт:**

> Прочитай тех задание в формате html. Начни реализовывать проект с моковыми данными. Архитектура: основная папка проекта (testing), внутри — frontend и backend. Стек: React (фронт), Python + FastAPI (бэк). Стили базовые, но красивые.

**Что сделано:**

- Создана структура проекта (frontend + backend)
- Backend: FastAPI, Pydantic-модели, mock_data
- Frontend: React, 6 страниц, Sidebar
- Моковые данные: 8 сотрудников, 2 шаблона, 3 онбординга, 25 задач

## Этап 4. Ролевая модель и профили

> Объединяет работу над авторизацией, фильтрацией и навигацией по ролям.

**Промпты:**

> Нужна возможность зайти под разными профилями — менеджер видит все задачи, разработчик только свои. Сделать подобие авторизации (выбор профиля без логина/пароля).

> Сайдбар тоже должен меняться? Зачем новичку видеть кол-во активных онбордингов? Для новичков упростить, для руководителей — свою логику. Так же нужно сделать обновления мгновенно при смене профиля, для каждого сотрудника своя логика

**Что сделано:**

- ProfileSelector (выбор профиля без логина/пароля)
- 5 ролей: HR, Manager, IT, Mentor, Newcomer
- Бэкенд-фильтрация по `X-User-Id` (HR — всё, Manager — свой отдел, остальные — свои задачи)
- Ролевая навигация в Sidebar (разные пункты меню для каждой роли)
- Динамическая статистика в футере Sidebar
- Мгновенное обновление данных через `user?.id` в useEffect

## Этап 6. Чатбот и база знаний

> Объединяет улучшение чата, кликабельные ссылки и расширение базы знаний.

**Промпты:**

> Оптимизируем работу с AI чатом. Чтоб он чётко говорил — спрашиваю что нужно сделать по моим задачам, он сразу чётко отвечает. Проработать пайплайн с базой знаний и чатом. Маркдауны в сообщениях — чтобы не было звёздочек, был нормальный текст.

> Если в ответе упоминается задача сотрудника — прикреплять ссылку. Нажимаю — открывается модалка с подробной информацией.

**Что сделано:**

- База знаний: 8 → 15 документов (коммуникации, доступы, оценка, ДМС, встречи, Jira, удалёнка)
- react-markdown для рендеринга ответов
- Кликабельные ссылки: `[Задача](task:id)`, `[Имя](onboarding:id)`, `[Документ](knowledge:doc-id)`
- 3 модалки: TaskDetailModal, OnboardingDetailModal, KnowledgeModal
- Сохранение чата в localStorage по пользователю
- Ролевые подсказки (suggestions)

## Этап 7. Шаблоны онбординга (CRUD)

**Промпты:**

> Сделаем модалку для добавления шаблона больше и шире. Слева блок этапов, справа задачи.

> Настрой логику для создания шаблона — нажимаю «создать», вылетает alert, но шаблон не появляется. Так же нужна возможность удалять и редактировать.

**Что сделано:**

- Backend: `POST/PUT/DELETE /api/templates`
- Модалка 960px, двухколоночный layout (этапы + задачи)
- Единый компонент TemplateModal (создание/редактирование)
- Кнопки редактирования и удаления (только HR)

## Этап 8. Финальная доработка по ТЗ

**Промпт:**

> Проверь приложение на полное соответствие ТЗ — максимально душно. Сделай всё, чтобы ТЗ было готово полностью. Рекомендации: глобальное видение проекта, архитектура, README, RAG, Деплой.

**Что сделано:**

- `POST /api/onboardings` — создание онбординга из шаблона (HR выбирает шаблон → задачи назначаются автоматически)
- `POST /api/knowledge/upload` — загрузка документов + переиндексация RAG
- UI «Спросить AI» на странице базы знаний (ответ с источниками)
- Страница `/onboardings` подключена в роутер и сайдбар
- Статус `waiting` для задач будущих этапов
- Пересчёт прогресса и SLA при смене статуса задачи
- Защита эндпоинтов шаблонов/онбордингов (только HR)
- Обработка ошибок API (`res.ok`), исправлены title/manifest
- Собственный RAG pipeline (`rag.py`): chunking → TF-IDF scoring → top-K retrieval
- 15 документов → 86 чанков, 792 токена в словаре
- RAG-контекст в чате (экономия токенов)
- Docker + docker-compose для деплоя
- Подробный README с архитектурными диаграммами

## Системные промпты AI

### Чатбот (`CHAT_SYSTEM_PROMPT`)

- Краткие ответы (10–15 строк), Markdown
- Обязательные ссылки: `[Title](task:TASK_ID)`, `[Name](onboarding:ONB_ID)`, `[Doc](knowledge:DOC_ID)`
- Источники из базы знаний с DOC_ID
- Адаптация под роль пользователя
- Если не знает — честно сообщает, предлагает HR

### Навигатор по базе знаний (`KNOWLEDGE_SYSTEM_PROMPT`)

- Ответ только на основе документов
- Обязательное указание источника: `[Название](knowledge:DOC_ID)`
- Markdown форматирование
