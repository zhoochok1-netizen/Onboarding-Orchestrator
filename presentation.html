Проблема -> Решение

Онбординг ломается там, где нет единого центра управления

Когда HR, IT, руководитель и наставник работают разрозненно, новичок теряется, задачи зависают, сроки плавают, а процесс становится непрозрачным. Проект превращает хаотичный набор действий в управляемый digital flow.

01

Разрозненные участники

Каждая роль отвечает за свой кусок процесса, но без общей системы никто не видит полную картину адаптации.

02

Сроки и SLA теряются

Без общего трекинга легко пропустить дедлайны, а просрочки замечаются уже тогда, когда новичок столкнулся с проблемой.

03

Нужен умный оркестратор

Система должна не просто хранить задачи, а координировать роли, подсказывать следующий шаг и быстро отвечать на вопросы.

Architecture

Full-stack архитектура, которую удобно объяснять за одну минуту

Фронтенд на React 19, backend на FastAPI, role-based API, собственный RAG pipeline и AI-gateway через OpenRouter. Архитектура компактная, понятная и при этом выглядит как готовая база для масштабирования.

Frontend

React 19 + Router 7

SPA с role-based навигацией, dashboard-экраном, доской задач, knowledge base и AI-чата в едином стиле.

Backend

FastAPI + Pydantic

REST API с валидацией, ролевой фильтрацией и понятными эндпоинтами для dashboard, задач, шаблонов и документов.

AI

OpenRouter gateway

Один интерфейс к моделям и гибкая смена провайдера через `.env`, без переписывания логики приложения.

Deploy

Docker + static frontend

Проект запускается просто и чисто: контейнеризированный backend, отдельная сборка frontend и готовность к деплою.
